# Principed d'organisations

1. Organisation modulaire du projet
Le projet est structuré en modules indépendants
2. Utilisation des branches et des Pull Requests
Chaque membre travaille sur sa propre branche
3. Intégration continue et code stable
La branche main contient uniquement du code fonctionnel et validé. Après chaque merge, toute l’équipe synchronise avec git pull pour garantir une version stable et cohérente du projet.
